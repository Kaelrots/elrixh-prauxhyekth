# 정의·위계(확정)

- **성권주(= 섹터)**: 둘 이상 **성계**를 하나의 **권역**으로 묶은 **주(州)** 급 상위 행정단위. 엘루닉어: **ryutz-zekamen-stakorh**. 여기서 `stakorh=주`, `zekam=조계(권역)`, 성계는 문서들에서 별도 천문 단위로 이미 쓰임.
    
- **위계**: 행성(…령) < 성계 < **성권주(섹터)** < 국가. 행성 내부 행정은 계속 **…령** 체계 사용, 성권주는 **성간** 행정단위로 분리.
    

# 경계·편성 규칙(제안)

- **편성 원칙**: 하이퍼레인/관문망의 **생활권 연속성**을 기준으로 2개 이상 성계를 묶어 설정.
    
- **예외**
    
    - **회랑 성권주**: 비연속 성계를 항행 회랑으로 묶는 특례.
        
    - **보호 성권주**: 차원류·성역 보존지 위주(거주 제한).
        
- **분구/합구**: 관문망 재배치·인구/생산 임계치 변화 시 의회 또는 황제령으로 개편. (주(州) 단위 상위로서 ‘주’ 개념과 호환됨)
    

# 명명·코드(운영안)

- **공식 명칭**: `<주요 성계명> 성계권역주` / 엘루닉어 병기: `ryutz-zekamen-stakorh <Toponym>`.
    
- **축약형**: **ryutzekorh**(성권주 약어) — UI·지도·코드에 사용.
    
- **코드 포맷**: `SEC-<SYS1>-<SYS2>-…` (3개 초과 시 `SEC-<SYS1>+n`).
    
    - 예: `SEC-RUA-AXN`(류아블론츠·AXN 성계 묶음), `SEC-RUA+3`.
        
- **문장 예시**: “엘리시움 신성제국 연방, **류아블론츠 성계권역주** 설치(섹터: `SEC-RUA+3`).”  
    주(州) 용례와 병치해도 무리 없음(헌문·법률에서 ‘주’가 상위 행정 단위 어휘로 이미 사용됨).

# 1) 정의를 딱 박자

- **행성(Planet)**: 자연·거주 단위. 내부 행정은 _직할령/자치령 등 ‘…령’_ 유지.
    
- **성계(Star system)**: 항성+행성군의 천문·항행 단위.
    
- **국가(Polity)**: 연방·제국·공화국 등 최상위 주권체.
    

> UI/지도: 긴 공식명(**성계권역주**)은 본문/법령에, 짧은 **“섹터”**는 실무·라벨에.

# 2) 예외·확장 규칙(미리 약속해두면 편함)

- **수도 성권주**: 국가 수도가 속한 성권주는 “특별성권주(수도)”로 표기(세율/치안/관문 우선권).
    
- **직할 성권주**: 황실·연방이 직접 통치하는 성권주.
    
- **공동 성권주**: 두 국가가 조약으로 공동관리(항로·관문 거점 중심).
    
- **보호 성권주**: 미거주·보존·금지구역 중심(차원류/모노리스 위험지대 포함).
    
- **복수 성계 성권주**: 항성 간 거리가 짧거나 관문망이 단일 생활권일 때만 승인.
    

# 3) 코드/표기 예시

- **국가**: 엘리시움 신성제국 연방 `POL-EHEF`
    
- **성권주(섹터)**: 류아블론츠 성계권역주 `SEC-RUA` _(= ryutz-zekamen-stakorh Ryuablontz)_
    
- **성계**: 류아블론츠 성계 `SYS-RUA`
    
- **행성**: 라리셴베르크 `PLN-LRS`
    
- **행성 행정구**: 황제직할령 제도 라흐 `REG-RAH`
    

계층 표기는 이렇게 직렬화하면 깔끔해:

`POL-EHEF / SEC-RUA / SYS-RUA / PLN-LRS / REG-RAH`

원하면 이 규칙을 **짧은 헌장 조문**(정의·예외·명명·코드 규칙 1페이지)으로 바로 써서 붙여줄게.

# 역사사건 메타데이터 체계화 v1.1

> 적용 결정(사용자 선택 반영)
>
> - (1) 사건의 종류는 `domains`가 아니라 별도 필드(`event_type/event_kind`)로 분리
> - (2) 도메인 상세 운용은 `domains_detail` **B안**(현행 1단계 도메인 + 필요 시 2·3단계 보조) 기준
> - (3) 기타 결정사항은 **A안** 기준(기존 캔버스의 A안 선택 항목 유지)

---

## 0. 설계 원칙

- **event\_type = 무엇이 벌어졌나(행위/사건의 “종류”)**
- **domains = 무엇이 영향받았나(영향 영역)**
- **actors = 사건 안에서 누가 어떤 역할(role)로 참여했나**
- **impact/assessment/reliability/cause\_phases/links\_index**는 “검색·분석·재사용”을 위한 별도 축

## 2) 사건의 종류(event\_type / event\_kind) — domains와 분리

### 목적

- domains는 “영향 영역”으로 유지
- 사건의 ‘행위/종류’는 별도 필드로 고정하여 분류 혼선을 제거

### 필드 권장

- `event_type`: 1\~2개(대표 1 + 보조 1)
- `event_type_code`: 코드 기반(3단계 권장)
- `event_type_label`: 사람이 읽는 표제

> NOTE: 현재는 코드 사전은 “추후 확장”으로 두고, 우선 label 중심으로 운용해도 됨.

### 9-1. event\_type와 domains의 완전 분리(강제)

- domains는 **영향 영역** 전용
- “무슨 사건인가”는 `classification.event_type`로 고정

### 9-2. EventSeries(사건군/시리즈) 도입

- 전쟁/혁명/대붕괴/장기 침식은 **시리즈 문서 1개 + 하위 사건 n개** 구조 권장
- `classification.event_series_ref`로 연결

### 9-3. 시간 불확실성(근사/불확실/범위) 1급 시민화

- 기존 세계관 달력 표기는 유지
- 추가로 `time.time_edtf`(불확실/근사/범위 허용 문자열)를 둬서 정렬·검색·표준화에 사용

### 9-4. 프로버넌스(provenance) 구조화

- reliability와 별개로, “이 서술이 어디서 왔나/어떤 편집·재서술을 거쳤나”를 `provenance`에 기록
- retcon(재서술) 사건에서 특히 강력

### 9-5. 장소(place)를 권위 레코드(Authority)로 승격

- 사건 문서에는 `place.place_ref = [[지역 문서]]` 링크를 기본으로
- 지역 문서는 지명 이명/행정단위 변천/좌표(선택) 등을 담는 권위 레코드 역할

### 9-6. actors도 권위 레코드(Authority)로 분리

- 사건 문서 = 사건 내 role만
- 행위자 문서(국가/기관/가문/개인) = 이명/존속기간/변천/관계망 등 “행위자 자체 메타”

# v1.1 추가 영역 세부 사전(초안)

> 아래는 \*\*v1.1에서 새로 추가한 영역(EventSeries / 시간 불확실성(time\_edtf) / provenance / 권위 레코드(place·actors))\*\*에 대해, 실제로 사건 문서에 바로 꽂아 쓸 수 있도록 **세부 분류 + 예시 + 복붙용 템플릿**까지 정리한 블록이다.

## 0) 사건 종류(event\_type) 코드 사전(확장)

### event\_type의 위치(핵심)

- **event\_type = “무슨 일이 벌어졌나(행위/형태)”**
- **domains = “영향을 받은 영역(정책/사회/경제 등)”**
- 따라서 event\_type은 원칙적으로 **primary 1개**를 두고, 필요하면 secondary를 0\~n개 둔다.

### 코드 형식(권장)

- `ET-ROOT.SUB.DETAIL` (문자열)
- ROOT는 굵게 유지(확장 가능), SUB/DETAIL은 필요할 때만 붙인다.

#### YAML에서의 기본 형태(복붙용)

```yaml
event_type:
  primary:
    code: ""
    label: ""
    external:        # (선택) 현실 분류체계 매핑이 필요할 때만
      cameo: ""     # GDELT/CAMEO 같은 외부 코드
      acled: ""     # ACLED event/sub-event 등
      emdat: ""     # EM-DAT 재난 분류 키워드/코드
  secondary: []      # - code/label 객체를 넣어도 되고, 문자열만 넣어도 됨
```

```yaml

    kind: "dimensional_crisis"     # series.kind 사전
    phase: "ignition"              # series.phase 사전
    membership_role: "anchor"      # anchor / major / minor
    note: "차원균열이 공식 확인되어 연쇄 사태가 본격화되는 구간."  # 1문장
```

## 2) 시간 불확실성(time\_edtf)

### 설계 원칙(엘리스 프라우셰크트용)

- 네 달력 규칙(1년=48개월, 1개월=30일)을 감안하면, 현실의 ISO/EDTF를 “그대로” 강제하기보다 **(1) 작중 표기 문자열 + (2) 정렬용 키 + (3) 불확실성 메타** 3단으로 가는 게 제일 안전하다.
- 따라서 v1.1에서는 `time_edtf`라는 이름을 유지하되, 실제 값은 **엘리스 프라우셰크트 내부 표준**으로 운용한다.

## 3) provenance(서술 계보 / 편집 계통)

### provenance 목적

  - 재서술(retcon) 분기 를 **추적 가능**하게 만든다.

\| 필드 | 설명 | 예시 | |---|---| | kind | legacy / integrated / transcript / image / external\_reference | | ref | 문서 링크/식별자 | | reliability\_hint | 간단 힌트(“검열 가능” 등) |

#### edit\_chain 객체(추천)

\| 필드 | 설명 | 예시 | |---|---| | actor | 편집 주체 | "현카엘" / "AI" | | date | 날짜 | "2025-12-24" | | action | 변환/수정 유형 | "table\_normalize" / "romanization\_fix" | | note | 1문장 | "로마자 표기열 밀림 수정" |

#### YAML 템플릿(복붙용)

```yaml
provenance:
  narrative_type: "chronicle"
  source_bundle:
    - kind: "legacy"
      ref: "[[레거시_제1기_연표]]"
      reliability_hint: "편년사 요약이라 누락 가능"
    - kind: "integrated"
      ref: "[[제1기_사건_통합본]]"
      reliability_hint: "표준화 적용"
  edit_chain:
    - actor: "AI"
      date: "2025-12-24"
      action: "table_normalize"
      note: "사건 메타데이터 표 구조 표준화"
  divergence:
    status: "none"          # none / exists
    alt_refs: []             # ["[[B루트_동일사건]]"]
    note: ""
  notes: "통합본 기준으로 서술하되 레거시 편년사 누락 가능성을 감안함."
```

## 4) 권위 레코드(authority records): place / actors

### 목적

- 사건 문서에서는 `actors.name`이나 `지리적 위치`가 계속 반복되니까,
  - 표기 흔들림(한국어/엘루닉어 로마자/구명칭/별칭),
  - 상하위 포함관계(도시⊂주⊂문명권),

## 4-C) 사건 문서에서 authority 레코드 참조하는 방식(권장)

### actors 입력 예시(사건 문서)

```yaml
actors:
  - type: "국가"
    name: "카로타니아 연방제국"    # 표시용
    authority_ref: "AC-0200"       # 권위 레코드 참조(추천)
    role: ["피해"]
```

### place 입력 예시(사건 문서)

```yaml
place:
  where_text: "카로타니아 연방제국 전역"
  authority_refs: ["PL-0999"]      # ‘전역’같이 추상일 때도 대표 권역 ID를 하나 붙여두면 검색이 쉬움
  notes: "국가 전역 규모의 붕괴."
```

# 적용 체크리스트(짧게)

- 사건이 **연쇄/국면**이 있으면 `series`를 1줄이라도 붙인다.
- 시간이 애매하면 `time_edtf.certainty`를 `circa/inferred/disputed` 중 하나로 표시한다.
- 사건 서술이 어디서 왔는지 애매하면 `provenance.source_bundle`에 최소 1개는 적는다.
- 반복 등장하는 국가/도시/기관은 가급적 `authority_ref`를 붙인다.

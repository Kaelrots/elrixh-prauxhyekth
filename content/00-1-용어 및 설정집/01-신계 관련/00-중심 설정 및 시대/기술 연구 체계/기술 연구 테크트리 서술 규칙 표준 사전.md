# 0. 목적
-  본 문서는 <font style="font-weight:bold">제2기 이후 기술·연구 테크트리</font>를 “시대별 게이트(노드) + 선후관계(엣지)”로 운영하기 위한 <font style="font-weight:bold">서술 규칙/표준/사전</font>이다.
-  목표:
  - (1) 문서량이 증가해도 <font style="font-weight:bold">ID·표기·분류·관계가 흔들리지 않게</font> 한다.
  - (2) `A → B` 선후관계로 <font style="font-weight:bold">트리/그래프를 생성</font>할 수 있게 한다.
  - (3) 시대 인덱스(목차)와 게이트 노드(본문)를 분리해 <font style="font-weight:bold">중복/충돌을 방지</font>한다.

# 작업 프로세스(표준)

> [!important] 문서 역할(SSOT)
> - <font style="font-weight:bold">10-01 시대 인덱스</font>: 시대별 “게이트 목록/발급대장” (gate_id 발급은 여기서 선확정)
> - <font style="font-weight:bold">10-02 게이트 노드</font>: 게이트 1개 설명(왜 게이트인가/매핑/요약/연계)
> - <font style="font-weight:bold">10-03 엣지 레지스트리</font>: 관계(선후/배타/대체)의 <font style="font-weight:bold">단일 기준(SSOT)</font>  
> - <font style="font-weight:bold">10-04 상세 기술(선택)</font>: 세계관 핵심 기술만 백과사전급으로 분기

## 0-1. 표준 워크플로우(권장 순서)
1) <font style="font-weight:bold">시대 정의(시대 문서)</font>  
   - 시대 경계(기간) + 변곡점 축(생산/군사/통치/차원/대외)만 먼저 확정
2) <font style="font-weight:bold">10-01 시대 인덱스 작성(게이트 목록/발급)</font>  
   - `gate_id` 발급(불변) + 인링크 골격  
   - 최소 기입: `tree_map`, `sub_type(공통12)`, `TMC(·CODE 허용)`, `한 줄 효과`, `Ref 키워드`
3) <font style="font-weight:bold">10-02 게이트 노드 생성(코어부터)</font>  
   - 최소 스키마(YAML) + “왜 게이트인가” + 매핑 근거  
   - <font style="font-weight:bold">중요 기술</font>만 `detail_doc`로 <font style="font-weight:bold">10-04 상세 기술</font> 분기
4) <font style="font-weight:bold">10-03 엣지 레지스트리 작성(관계 원장)</font>  
   - 시대 내부(로컬) → 교차시대(점프) 순  
   - 교차시대 엣지는 `note` 1줄 이상 필수
5) (선택) <font style="font-weight:bold">검증/시각화 출력(코딩 영역)</font>  
   - 10-03을 입력으로 그래프 생성/사이클/중복/누락 자동검증

## 0-2. 작업 단위(추천: “코어 8개” 루프)
- ① 인덱스에 코어 8개 채움 → ② 노드 8개 생성 → ③ 엣지로 뼈대(10~30개) 잡기  
- 이후 “확장 후보”를 12~22개로 늘리면서 ①②③을 반복(얇게 순환)

## 0-3. 변경/유지보수 규칙(필수)
- `gate_id`는 <font style="font-weight:bold">발급 후 불변</font>(이름/설명은 바뀌어도 됨)
- 관계 충돌 시 우선순위:
  1) <font style="font-weight:bold">10-03 엣지 레지스트리</font>
  2) 10-02 게이트 노드(설명/맥락)
  3) 10-01 인덱스(목록)
- 삭제 대신 `status: deprecated`로 폐기(관계/노드 추적성 유지)
- 시대 점프(교차시대) 엣지는 <font style="font-weight:bold">근거(note/source) 없는 채로 금지</font>

## 0-4. 빠른 품질 점검(체크리스트)
- [ ] 10-01의 `gate_id`와 10-02/10-03의 ID가 1:1로 대응
- [ ] 모든 노드에 `tree_map` ≥ 1, `sub_type` 1개 지정
- [ ] 교차시대 엣지에 `note` 존재
- [ ] `alt_path`와 `mutex` 혼용 없음
- [ ] 의도치 않은 사이클(특히 `prereq_hard`) 없음

# 3. 식별자(Identifier) 규칙
## 3-1. 시대 코드(era)
-  형식: `2-01`, `2-02`, … `2-99`
-  의미: `2기-시대번호`
-  예시: `2-03` = 제2기 03시대

## 3-2. 게이트 ID(gate_id)
-  형식: `G2-<era>-<nnn>`
  - `<era>`: `01`～`99` (시대번호 2자리)
  - `<nnn>`: `001`～`999` (3자리 연번)
-  예시:
  - `G2-03-001`
  - `G2-03-014`

## 3-3. 표 No(번호) 원칙
-  시대 인덱스 표의 `No`는 <font style="font-weight:bold">보기용</font>이다.
-  게이트를 추가/삭제하며 번호가 변경되어도 <font style="font-weight:bold">gate_id는 절대 변경하지 않는다.</font>

## 3-4. 이름(게이트명) 원칙
-  게이트명은 바뀔 수 있다(명칭 개편/번역 수정 등).
-  그러나 링크/관계/키는 <font style="font-weight:bold">항상 gate_id</font>를 기준으로 한다.

> [!important] 불변 규칙
> <font style="font-weight:bold">gate_id는 한 번 발급하면 영구 불변</font>이다.

<hr class="hr-thick-2">

# 4. TMC(성숙도·단계) 표기 규칙
## 4-1. 기본 형식
-  형식: `<ERA>-<TRL>`
-  ERA(시대축): `ANC / MED / EMO / IND / MOD / NFU / FUT / SPC`
-  TRL(성숙도): `1`～`9`

## 4-2. 기간/범위 표기
-  범위형: `MOD-7～9` 처럼 표기한다.
-  문서 전체에서 `~`는 사용하지 않고 <font style="font-weight:bold">`～`</font> 로 통일한다.

## 4-3. (선택) 특수 코드(Special Code)
- 특수 코드는 `·`로 결합한다.
- 표기 권장: `<ERA>-<TRL>·CODE`
  - CODE: `LST / ARC / TRN`
- 예시:
  - `MOD-7～9·LST` (현대 후기급으로 ‘추정’되나 전승 단절)
  - `NFU-5·ARC` (근미래 중기급 + 신비)
  - `SPC-2·TRN` (우주 시대 분류 + 초월)
- 주의: “실전”은 실전(實戰)이 아니라 <font style="font-weight:bold">실전(失傳)</font> 의미로 사용한다.

# 6. 연구트리 3단계 매핑 규칙(tree_map)

## 6-0. 세부 유형(sub_type) 전제(공통 12종)
- 세부 유형은 트리마다 따로 만들지 않고, <font style="font-weight:bold">공통 12종으로 통일</font>한다.
- 원칙: 노드는 <font style="font-weight:bold">`트랙(3단계)` + `세부유형(1～12)`</font> 조합으로 구성된다.
- 공통 12종:
  1. 이론/원리
  2. 계측/표준
  3. 공정/제조
  4. 인프라/네트워크
  5. 제도/법규
  6. 조직/운영
  7. 종교교리/규범
  8. 군사교리/훈련
  9. 통제/보안
  10. 대외/접촉
  11. 시험/검증(V&V)
  12. 모델/시뮬레이션/소프트웨어(SIM)
- 표기(권장): `sub_type: "<번호>. <이름>"` (예: `sub_type: "2. 계측/표준"`)

## 6-1. 매핑 필수
-  <font style="font-weight:bold">모든 gate_node는 최소 1개의 3단계 매핑을 가져야 한다.</font>
-  시대 인덱스(era_gate_index)에도 동일 매핑을 요약 기입한다.

## 6-2. 복수 매핑 허용 기준
-  복수 매핑은 허용하되, 아래 중 하나를 충족해야 한다:
  - (A) 하나의 게이트가 실제로 2개 이상의 분야를 <font style="font-weight:bold">동시에</font> 개방한다.
  - (B) ‘핵심 매핑 1개 + 보조 매핑 1개’로 설명 가능하다.

## 6-3. 표기 형식
-  표기: `3-XX → (하위 트랙명)`
-  복수 매핑 시: 줄바꿈/리스트로 나열

## 6-4. 매핑 검증 원칙
-  존재하지 않는 `3-XX` 코드를 임의로 사용하지 않는다.
-  애매하면, 게이트 노드 문서의 “매핑 근거”에 <font style="font-weight:bold">왜 그 트랙인지</font> 2～5줄로 명시한다.

# 7. 선후관계(Edge) 규칙
## 7-1. 관계의 단일 키
-  관계는 항상 `gate_id`로만 표현한다.
-  게이트명으로 관계를 정의하지 않는다.

## 7-2. Hard/Soft 선행조건
-  `Hard`: 없으면 <font style="font-weight:bold">절대</font> 성립/운용이 불가능한 선행
-  `Soft`: 강하게 권장되나, 우회/대체 경로가 가능한 선행

## 7-3. 엣지 레지스트리(rel_type) 사전

## 7-4. weight 규칙(선택)
-  `weight`는 0.0～1.0 사이로 둔다.
  - 1.0 = 거의 필수 수준
  - 0.6 = 강한 권장
  - 0.3 = 느슨한 연계

## 7-5. 시대 점프 엣지
-  `2-01 → 2-04`처럼 시대를 크게 건너뛰는 엣지는 허용하되,
-  엣지 레지스트리의 `note`에 <font style="font-weight:bold">왜 점프하는지</font> 반드시 1줄 이상 적는다.

## 7-6. 사이클(순환) 규칙
-  의도치 않은 사이클은 금지.
-  의도된 순환(상호강화 루프)을 표현해야 한다면:
  - `synergy`로 표현하고,
  - `prereq_*`로 사이클을 만들지 않는다.

<hr class="hr-thick-2">

# 8. 문서별 필수 요소(최소 스키마)
## 8-1. 시대 인덱스(era_gate_index)
-  반드시 포함:
  - 시대 코드(era), 시대명(era_name)
  - 게이트 표(최소 core 8개)
  - 각 행에: `gate_id`, `인링크`, `tree_map`, `TMC`, `한줄 효과`

## 8-2. 게이트 노드(gate_node)
-  반드시 포함:
  - `gate_id`, `gate_name`, `era`, `tmc`, `tree_map`
  - `prereq.hard[]`, `prereq.soft[]`, `unlocks[]` (없으면 빈 배열)
  - 본문 섹션: `개요 / 왜 게이트인가 / 매핑 근거 / 선행 / 후속`

## 8-3. 엣지 레지스트리(gate_edges)
-  반드시 포함:
  - `from_gate`, `to_gate`, `rel_type`
  - (권장) `weight`, `note`

<hr class="hr-thick-2">

# 9. 금지·권장 규칙
## 9-1. 금지
-  gate_id 재사용/재발급(중복) 금지
-  게이트명을 키로 삼는 관계 정의 금지
-  시대 인덱스에 관계 본문을 길게 서술 금지
-  존재하지 않는 3단계 코드 임의 생성 금지

## 9-2. 권장
-  게이트 노드에는 “경계선(포함/비포함)”을 1줄이라도 적기
-  시대 점프 엣지는 반드시 `note`로 설명
-  오버레이는 정말 필요할 때만

<hr class="hr-thick-2">

# 10. 품질 점검 체크리스트
## 10-1. 신규 게이트 추가 시
- [ ] gate_id 유일성 확인
- [ ] 시대 인덱스 표에 `gate_id + 인링크` 반영
- [ ] gate_node에 `tree_map` 최소 1개 존재
- [ ] prereq/unlocks가 있으면 엣지 레지스트리에도 반영
- [ ] “왜 게이트인가”가 시대 변곡점과 연결되어 있음

## 10-2. 트리(관계) 점검
- [ ] 의도치 않은 사이클 없음
- [ ] 허브 과집중(병목)이 의도인지 확인
- [ ] 시대 점프 엣지에 note 존재
- [ ] mutex/alt_path를 혼동하지 않음

<hr class="hr-thick-2">

# 11. 변경·버전 관리 규칙
-  본 문서의 변경은 <font style="font-weight:bold">문서버전</font>을 올리고, `최근 변경 요약`에 1줄 기록한다.
-  규칙 변경이 개별 문서에 영향을 줄 경우:
  - (권장) “영향 범위(예: 게이트 노드 전체)”를 변경 요약에 함께 적는다.
